Cette section comprend plusieurs ressources comme expliqué ci-dessous :

Configuration de l'authentification

Une définition de configuration d'authentification individuelle, qui est utilisée par les clients déployés dans ${build.fullName} et d'autres ressources pour l'authentification lors de l'établissement d'une connexion à distance.

Contexte d'authentification

Une définition de contexte d'authentification individuelle, qui est utilisée pour fournir un contexte ssl et une configuration d'authentification lorsque des clients déployés vers ${build.shortName} et d'autres ressources établissent une connexion distante.

Stores

Magasin de clés : définition d'un magasin de clés ou d'un magasin de confiance, y compris le type de magasin de clés, son emplacement et les informations d'identification permettant d'y accéder. Le magasin d'informations d'identification permet de conserver des alias pour les informations sensibles telles que les mots de passe pour les services externes.

SSL

Configurer le côté client et le côté serveur des connexions SSL. Une définition du gestionnaire de clés pour créer la liste des gestionnaires de clés utilisée pour créer un contexte SSL. Autres ressources disponibles : Propriétés de sécurité, Chargeur de fournisseurs, Fournisseurs agrégés.

Security Domains

Un domaine de sécurité est la représentation d'une politique de sécurité qui est soutenue par un ou plusieurs domaines de sécurité et un ensemble de ressources qui effectuent des transformations.

Logs

Audit loggers qui persistent sur les fichiers et syslog.
