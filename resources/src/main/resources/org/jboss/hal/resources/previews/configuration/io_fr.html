Le sous-système IO définit les workers XNIO et les pools tampons utilisés par d'autres sous-systèmes, tels que Undertow et Remoting.

Worker

Les workers sont des instances de workers XNIO. Une instance de worker XNIO est une couche d'abstraction pour les API Java NIO, qui fournissent des fonctionnalités telles que la gestion des IO et des threads de travailleur ainsi que la prise en charge de SSL. Par défaut, ${build.shortName} fournit un seul worker appelé default, mais il est possible d'en définir d'autres.

Pools de mémoire tampon

Les pools de mémoire tampon sont des instances de mémoire tampon NIO groupées.

Important :
La modification de la taille de la mémoire tampon a un impact important sur les performances de l'application. Pour la plupart des serveurs, la taille idéale est généralement de 16 Ko.
