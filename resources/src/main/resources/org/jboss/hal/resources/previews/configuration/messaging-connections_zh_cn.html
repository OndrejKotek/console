包含以下功能的配置：

  • Acceptor 和 Connector
  • Connector 服务
  • Connection Factories （池）

Acceptor 和 Connector

一个接受者（acceptor）定义了 ${build.shortName} 嵌入的消息服务器接受哪一种连接。您可以为每个服务器定义任意数量的接受者。

connector 定义了如何连接到一个内建的 ${build.shortName} 消息服务器，它被客户系统用来进行连接。

如需了解更多与 acceptor 和 connector 相关的信息，请参阅 Acceptors and Connectors section。

Connector 服务

connector 服务的功能是集成外部组件和 Apache ActiveMQ Artemis 来发送和接收消息。

连接工厂

默认情况下，${build.shortName} 消息子系统提供了 InVmConnectionFactory 和 RemoteConnectionFactory 连接工厂，以及 activemq-ra 池的连接工厂。

InVmConnectionFactory 引用 in-vm-connector，它可以用来在客户端和服务器在同一 JVM 中运行时发送和接收消息。RemoteConnectionFactory 引用 http-connector，它可以被用来在客户端和服务器在不同 JVM 中运行时通过 HTTP 发送和接收消息。

池化连接工厂允许您配置集成 ActiveMQ Artemis 资源适配器的入站和出站连接器。如需了解更多与配置 pooled-connection-factory 连接到远程 ActiveMQ Artemis 服务器的信息，请参阅使用集成资源适配器进行远程连接。
