Le sous-système EE fournit des fonctionnalités communes à la plateforme Jakarta EE, telles que les utilitaires EE Concurrency et l'injection @Resource. Le sous-système est également responsable de la gestion du cycle de vie des déploiements d'applications Jakarta EE, c'est-à-dire des fichiers .ear.

La configuration du sous-système EE peut être utilisée pour :

  • Permet de personnaliser le comportement de déploiement des applications Jakarta EE.
  • Gérer les modules globaux : un ensemble de modules JBoss qui seront ajoutés en tant que dépendances au module JBoss de chaque déploiement Jakarta EE. Ces dépendances permettent aux déploiements Jakarta EE de voir les classes exportées par les modules globaux.
  • Gérer les utilitaires de concurrence EE : ils ont été introduits pour faciliter l'écriture d'applications Jakarta EE multithreadées. Les instances de ces utilitaires sont gérées par ${build.fullName} et la configuration correspondante est fournie par le sous-système EE.
