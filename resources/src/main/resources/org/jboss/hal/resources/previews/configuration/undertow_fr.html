Le sous-système Undertow vous permet de configurer les paramètres du serveur web et du conteneur de servlets. Il implémente la spécification Java Servlet 3.1 ainsi que les websockets et prend en charge HTTP Upgrade et l'utilisation de gestionnaires non bloquants à haute performance dans les déploiements de servlets. Le sous-système Undertow a également la capacité d'agir comme un proxy inverse de haute performance qui prend en charge mod_cluster.

Dans le sous-système Undertow, il y a six composants principaux à configurer :

  1. Paramètres globaux
  2. Caches tampons
  3. Serveur
  4. Conteneur de servlet
  5. Filtres
  6. Handlers

Important
Le sous-système Undertow s'appuie également sur le sous-système IO pour fournir des workers XNIO et des pools de tampons. Le sous-système IO est configuré séparément et fournit une configuration par défaut qui devrait offrir des performances optimales dans la plupart des cas.
